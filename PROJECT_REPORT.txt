1. PROJECT OVERVIEW
-------------------
The AI Arbitrage Bot has evolved into an "Agentic" high-frequency trading 
system. It no longer relies solely on historical patterns but now utilizes 
Large Language Models (LLMs) to verify market conditions in real-time before 
executing capital-at-risk trades between Binance and Bybit.

2. THE "PIXEL" INTERFACE (VISUALS & UX)
---------------------------------------
The React + Vite dashboard serves as the Mission Control Center:
- Secure Session Control: Integrated with a custom "Local Operator Vault" 
  for authentication.
- Live Telemetry: Streams real-time ticker data, AI confidence scores, 
  and system logs directly from the Python core via WebSockets.
- Safety Dashboard: Visual indicators for AI cooldowns, execution 
  thresholds, and connection health.

3. BACKEND ARCHITECTURE (INFRASTRUCTURE)
----------------------------------------
The system operates on a dual-engine multi-threaded architecture:
- API Gateway (`api.py`): A FastAPI-based secure vault managing 
  operator credentials and session tokens with 32-byte hex encryption.
- Execution Core (`main.py`): Orchestrates the parallel running of the 
  web server and the trading engine.
- Persistent Vault: Uses `operator_vault.db` for security and `trades.db` 
  for fiscal transparency.

4. THE "AGENTIC" LOGIC (INTELLIGENT DECISION MAKING)
----------------------------------------------------
The "Brain" has been upgraded from a static GRU model to a Multi-AI Agent:
- Primary Intelligence: Google Gemini (2.5-Flash) analyzes live price spreads
  and exchange conditions to provide an "EXECUTE" or "REJECT" verdict.
- Fail-Safe Fallback: If Gemini hits rate limits or latency issues, the 
  system automatically switches to Groq (LLaMA 3.1) for a secondary opinion.
- Confidence Scoring: The AI provides a 0-100 confidence level, which acts 
  as the final gate before order submission.

5. REAL-WORLD EXECUTION ENGINE
------------------------------
- Real Market Orders: The bot performs actual "Market Buy" executions on 
  the Binance Testnet using the CCXT library.
- Hybrid Simulation: To counter network restrictions, the system employs 
  a "Hybrid Leg" strategy—executing real trades on Binance while 
  simulating the Bybit leg when API connectivity is blocked.
- Trade Persistence: Every successful cycle is logged to a local SQLite 
  database, capturing order IDs, average prices, and net costs.

6. RISK & STABILITY
-------------------
- Automatic Cooldowns: Implements a 60-second "AI thinking" cooldown to 
  prevent API exhaustion and "over-trading" in volatile markets.
- Error Trapping: Advanced handlers for "429 Quota" errors and "API Key" 
  failures, ensuring the bot aborts safely rather than crashing.

7. MODEL STATUS & SYSTEM ANALYSIS
---------------------------------
- [COMPLETED] Multi-AI Agent: Gemini + Groq fallback system is fully LIVE.
- [COMPLETED] Local Operator Vault: Secure registration/login API is active.
- [COMPLETED] Binance Real Execution: Market order logic is operational.
- [IN PROCESS] Live Telemetry: Refinement of the WebSocket sync between 
  `main.py` and the React frontend.
- [PENDING] Portfolio Scaling: Moving from static 0.01 BTC trades to

STRENGTHS:
- High Intelligence: Dual-LLM confirmation minimizes "false positive" trades.
- Robust Auth: Hashed password security and unique session tokens.
- Safety First: Comprehensive testnet sandboxing and error-recovery logic.

WEAKNESSES:
- API Dependency: Highly reliant on external AI providers (Gemini/Groq).
- Initial Friction: Requires valid API keys for execution (currently 
  being configured).

8. CONCLUSION
-------------
The AI Arbitrage Bot represents the next generation of algorithmic trading, 
where deep learning is augmented by LLM reasoning. By combining real 
execution capabilities with a secure operator interface, the system is 
well-positioned for transitioning from a sandbox environment to live 
market participation.
