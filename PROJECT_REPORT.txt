1. PROJECT OVERVIEW
-------------------
ArbPro is an enterprise-grade AI-powered cryptocurrency arbitrage bot that detects
and exploits price discrepancies between Binance and Bybit exchanges on the BTC/USDT
pair. The system is fully modularized with a component-based React architecture, a
FastAPI backend with real-time data oracles, and an advanced MULTI-LAYER LLM CORE
providing trade validation, sentiment analysis, risk assessment, and strategic guidance.

2. ARCHITECTURE OVERVIEW (v7.0 - ENHANCED WITH LLM STRATEGY)
-------------------------------------------------------------

  [A] BACKEND CORE (Python / FastAPI / CCXT)
      - api.py              : Central API & Secure WebSocket engine
      - main.py             : Unified application entry point
      - predictor.py        : GRU-based sequence prediction (10-step forecasting)
      - execution/trader.py : Async trade execution with proxy rotation & fallback

  [B] LLM INTEGRATION LAYER (v7.0 - NEW COMPREHENSIVE STRATEGY)
      TIER 1 - TRADE DECISION ENGINE (llm/ai_agent.py - ENHANCED)
      ========================================================
      - Primary LLM: Google Gemini 2.0 Flash
        * Analyzes spread, price volatility, and execution timing
        * JSON-structured decision output: {decision, reason, confidence}
        * Context awareness: Previous 5 trades + market conditions
      
      - Fallback LLM: Groq LLaMA 3.1 (Ultra-fast)
        * Activates if Gemini fails or timeout > 2s
        * Same JSON schema for compatibility
        * Sub-50ms response guarantee
      
      - Safety Gate: Risk Veto System
        * Rejects trades if position sizing exceeds risk threshold
        * Analyzes cumulative exposure across time windows
        * Implements Kelly Criterion for optimal bet sizing

      TIER 2 - MARKET INTELLIGENCE ENGINE (llm/market_analyst.py - NEW)
      ==================================================================
      - Trend Analysis: 
        * Analyzes 50-candle historical patterns
        * Identifies market regime (trending/sideways/volatile)
        * Generates sentiment scores (-1.0 to 1.0)
      
      - Volatility Forecasting:
        * Uses LLM to interpret VWAP, Bollinger Bands, ATR
        * Predicts optimal trade window duration
        * Suggests position sizing based on volatility

      TIER 3 - STRATEGY OPTIMIZER (llm/strategy_advisor.py - NEW)
      ===========================================================
      - Performance Analysis:
        * Reviews historical trades and identifies patterns
        * Extracts meta-strategy insights: "Best wins occur at X UTC time"
        * Recommends threshold adjustments based on market regime
      
      - Portfolio Rebalancing:
        * Analyzes total profit/loss across defined periods
        * Recommends dynamic threshold changes
        * Suggests rest periods to avoid over-trading

      TIER 4 - FRONTEND CHAT ASSISTANT (Frontend/AIAssistant.tsx - ENHANCED)
      ====================================================================
      - Real-time Trading Q&A:
        * "Why did bot reject that trade?" → LLM explains via WebSocket
        * "What's the market sentiment?" → Retrieves from market_analyst
        * "Optimize my trading strategy" → Calls strategy_advisor
      
      - Confidence Transparency:
        * Shows decision confidence % from LLM
        * Alerts if multiple LLM layers disagree
        * Logs all rejections with detailed reasoning

  [C] FRONTEND (Modular React / TypeScript / Recharts)
      - Dashboard.tsx       : Main orchestration layout
      - components/dashboard/
          - Header.tsx      : Status, controls, LLM indicators
          - PortfolioMetrics.tsx : Real-time wealth tracking

  4.3 PHASE 3: RISK MANAGEMENT & POSITION SIZING (Week 3)
      ═════════════════════════════════════════════════════

      Step 3.1: Implement Kelly Criterion Sizing
      ──────────────────────────────────────────
      
      Formula:
      f* = (bp - q) / b
      where:
      • f* = optimal fraction of capital to bet
      • b = odds ratio (profit per $1 risked)
      • p = win probability
      • q = 1 - p (loss probability)
      
      Application:
      └─ Calculate from previous 50 trades
      └─ Adjust based on confidence < 70% → cap at 30% Kelly
      └─ Dynamic check before every trade

      Step 3.2: Risk Veto System
      ──────────────────────────
      
      Checks before execution:
      1. Cumulative position risk (rolling 24h) < 2%
      2. Single trade risk < 0.5% of portfolio
      3. Correlation check (avoid multiple BTC pairs)
      4. Circuit breaker: If 3 losses in row → cooldown 5min
      5. Daily max loss: If -1.5% → HALT until reset

      Step 3.3: Add to execution/trader.py
      ────────────────────────────────────
      
      New Method:
      • async execute_trade_with_risk_check(symbol, size) → bool
        └─ Calls risk_engine.validate_position()
